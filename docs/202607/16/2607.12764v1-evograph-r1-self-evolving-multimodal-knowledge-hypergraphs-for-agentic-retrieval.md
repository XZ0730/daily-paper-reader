## 摘要
检索增强生成(RAG)已成为将多模态大语言模型(MLLMs)扎根于外部知识的关键范式。最近的GraphRAG方法引入了结构化的实体-关系图来改进检索与推理。然而，它们仍受限于将知识图谱视为离线构建且单次查询的静态数据结构。这种静态范式与知识密集型推理的交互性、迭代性本质不相符，产生了三个瓶颈：(i) 以文本为中心的分割阻碍了跨模态推理，(ii) 冻结的结构无法纳入新证据或纠正错误，以及 (iii) 僵化的单次检索缺乏自适应精化。为克服这些限制，我们提出了EvoGraph-R1，一个自演化的GraphRAG框架，将知识图谱重新概念化为通过智能体交互形成的动态环境。我们将检索形式化为一个马尔可夫决策过程(MDP)，其中智能体观察图状态并执行动作来查询(GraphRetrieve)、扩展(WebSearch)、精化(GraphEdit)或终止(Answer)推理。这些动作重塑超图结构并生成反馈信号指导后续演化。通过这一闭环，超图通过整合新证据、纠正错误和精化结构来支持多跳推理而不断演化。在多模态VQA和文本QA基准上的实验表明，与现有RAG基线相比，在准确性、覆盖度和可追溯性方面有显著提升，确立了自演化知识图谱作为跨模态的基础范式。

## Abstract
Retrieval-augmented generation (RAG) has emerged as a critical paradigm for grounding Multimodal Large Language Models (MLLMs) in external knowledge. Recent GraphRAG methods introduce structured entity-relation graphs to improve retrieval and reasoning. However, they remain limited by treating knowledge graphs as static data structures built offline and queried in a single pass. This static paradigm misaligns with the interactive, iterative nature of knowledge-intensive reasoning, creating three bottlenecks: (i) text-centric fragmentation that impedes cross-modal reasoning, (ii) frozen structures unable to incorporate new evidence or correct errors, and (iii) rigid single-pass retrieval without adaptive refinement. To overcome these limitations, we introduce EvoGraph-R1, a self-evolving GraphRAG framework that reconceptualizes knowledge graphs as dynamic environments shaped through agent interactions. We formulate retrieval as a Markov Decision Process (MDP) where the agent observes the graph state and executes actions to query (GraphRetrieve), expand (WebSearch), refine (GraphEdit), or terminate (Answer) the reasoning. These actions reshape the hypergraph structure and generate feedback signals that guide subsequent evolution. Through this closed loop, the hypergraph evolves by integrating new evidence, correcting errors, and refining structure to support multi-hop reasoning. Experiments on multimodal VQA and text QA benchmarks demonstrate substantial improvements over existing RAG baselines in accuracy, coverage, and traceability, establishing self-evolving knowledge graphs as a fundamental paradigm across modalities.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：现有的检索增强生成（RAG）与知识图谱增强生成（GraphRAG）方法将知识图谱视为**静态数据结构**，在离线阶段一次性构建，推理时仅执行单次查询。这种静态范式与知识密集型推理的**交互性、迭代性**本质相悖，导致三大瓶颈：
  - **文本中心分割**：将多模态证据简化为纯文本元组，丧失细粒度结构，阻碍跨模态推理。
  - **冻结结构**：无法在推理过程中纳入新证据、纠正错误或覆盖未见知识。
  - **僵化检索**：单次检索缺乏自适应精化，初始证据不足时无法调整策略或调用外部搜索。
- **整体含义**：论文重新定义了知识图谱的角色——从“被动数据结构”升维为“主动推理环境”。通过让知识图谱与智能体**协同演化**，实现一种**自演化检索范式**，从而显著提升多模态与纯文本知识密集型问答的准确性、覆盖度与可追溯性。

## 2. 论文提出的方法论
- **核心思想**：将多模态知识检索形式化为一个**马尔可夫决策过程（MDP）**，其中：
  - **环境状态**：动态的多模态知识超图 $G_t = (V_t, E_t)$，编码实体/关系/视觉对象及其连接、对齐等。
  - **智能体动作**：四种原子操作——**GRAPH RETRIEVE**（查询图内信息）、**WEB SEARCH**（接入外部搜索）、**GRAPH EDIT**（插入/更新/删除图元素）、**ANSWER**（终止并输出答案）。
  - **奖励**：综合考虑**结构正确性**、**答案语义匹配度**（F1式）和**执行效率**。
  - **策略**：通过强化学习训练动作选择策略 $\pi_\theta(a_t|s_t)$，最大化轨迹回报。
- **关键技术细节**：
  - **多模态超图构建**：从图文语料中抽取n元关系超边；视觉内容通过生成场景描述并提取视觉实体/关系，并锚定到图像节点；最终通过实体解析融合为统一超图，并用多模态编码器（GME）离线索引。
  - **智能体引导的图演化**：每步动作后，超图依据演化策略 $\pi_{evolve}$ 发生转移。`GRAPH EDIT` 的三个子操作（Insert/Update/Delete）直接修改图结构，`WEB SEARCH` 将检索到的外部证据对齐后注入文本上下文，供后续演化使用。历史轨迹 $H_t$ 被记录以指导后续决策。
  - **强化学习优化**：采用Group Relative Policy Optimization (GRPO) 训练策略。轨迹级奖励 $R(\tau)$ 由三部分构成：
    $$R(\tau) = -\lambda \sum_{t} c(a_t) + R_{struct}(\tau) + \mathbb{1}[R_{struct}=1.0] \cdot R_{ans}(a_{ans})$$
    其中 $c(a_t)$ 为动作成本，$R_{struct}$ 为步骤合规性奖励，$R_{ans}$ 为答案正确性奖励。仅当结构奖励满分时才给予答案奖励，以确保推理过程规范性。
- **算法流程**：输入查询 $q$ 与初始超图 $G_0$；智能体循环执行动作 $a_t$，更新状态 $s_{t+1}$，直到触发 `ANSWER` 或达到最大步数；最终基于演化后的图 $G_T$ 生成答案。

## 6. 论文的主要结论与发现
- **性能全面领先**：EvoGraph-R1 在所有文本 QA 和多模态 VQA 基准上均取得最优结果。尤其在多模态 E-VQA 上（43.6%）显著超过 MMSearch-R1 (+6.7%) 和 MMKB-RAG (+7.7%)；在文本 2WikiMultiHopQA 上 F1 达 68.5%，比最强的 Graph-R1 高 +3.5%。

- **跨模态通用性**：框架可通过移除视觉组件自然适配纯文本任务，验证了“把知识图作为交互环境”这一范式在模态上的独立性。

## 7. 优点
- **范式创新**：首次将知识图谱建模为**动态进化环境**，通过 MDP 统一检索、推理与知识更新，突破了 GraphRAG 的静态瓶颈。
- **闭环自演化**：智能体通过“查询-搜索-编辑”闭环持续精化超图，使其与推理协同演化，有效解决了初始图的不完整性和噪声问题。
- **精细的超图设计**：采用超图而非普通图，支持 n 元关系；显式维护视觉锚点与跨模态对齐，保留了细粒度多模态语义。

- **可解释性与可追溯性**：动态演化过程记录了每步决策和结构变化，推理路径透明，可追溯证据来源。
