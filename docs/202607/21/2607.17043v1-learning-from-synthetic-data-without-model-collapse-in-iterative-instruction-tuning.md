## 摘要
模型崩溃是从合成数据中学习的核心挑战：随着后期的大语言模型（LLM）在越来越多模型生成数据上进行训练，由于覆盖范围变窄和累积偏差，性能可能会下降。现有工作主要研究如何约束这种退化。然而，在迭代模型进化中，更有意义的目标是确保每个后续模型都比其前身有所改进，这需要以可指导数据管理的粒度诊断崩溃。我们研究了合成数据在指令调优中自我改进的这一问题。我们表明，在此场景下的崩溃并非简单的均匀性能下降，而可能表现为能力的极化，即合成训练强化了已经擅长的技能，同时进一步削弱了较弱技能。受此观察启发，我们提出了 KITE（通过探索进行知识边界指令调优），一个两阶段框架，它将失败引导的数据生成与边界感知不确定性管理相结合。在多个数据集和多种开源 LLM 上的实验表明，KITE 比强合成数据基线带来更稳定的改进。

## Abstract
Model collapse is a central challenge in learning from synthetic data: as later-generation large language models (LLMs) are trained on an increasing proportion of model-generated data, performance can degrade due to narrowed coverage and accumulated bias. Existing work mainly studies how to bound this degradation. In iterative model evolution, however, the more meaningful objective is to ensure that each successive model improves over its predecessor, which requires diagnosing collapse at a granularity that is actionable for data curation. We study this problem in synthetic data self-improving for instruction tuning. We show that collapse in this setting is not simply uniform performance degradation, but can appear as a polarization of competence, where synthetic training reinforces already strong skills while further degrading weak ones. Motivated by this observation, we propose KITE (Knowledge-boundary Instruction Tuning via Exploration), a two-stage framework that combines failure-guided data generation with boundary-aware uncertainty curation. Experiments across several datasets and multiple open-source LLMs show that KITE yields more stable improvement than strong synthetic-data baselines.

- **关键发现**：在指令微调的自我改进中，崩溃并非整体的性能均匀下降，而是**能力的极化**——合成训练会强化模型原本已擅长的技能，同时进一步削弱其本就薄弱的技能（如数学代数、时间调度等）。这一发现为有针对性的数据构建提供了动机。

### 方法论：KITE 框架
KITE（Knowledge-boundary Instruction Tuning via Exploration）是一个两阶段数据构建框架，旨在让每一轮新增的合成数据都能为下一代模型提供有效的学习信号。

**第一阶段：失败引导的候选指令生成**
- **弱点画像**：利用教育心理学中的 **DINA 认知诊断模型**评估当前模型在训练集上的掌握程度。模型为每条训练数据关联一个二进制技能需求向量（Q矩阵），通过分析回答正误模式，估计各技能的**后验掌握度**，并将低掌握度的技能转化为自然语言形式的弱点描述。
- **排序噪声注入**：针对每个弱点，使用设定的提示模板生成新指令。为鼓励探索，防止模型只在高概率区域生成模板化样本，采用**秩基扰动解码**：
  - 计算每个token的排序 `rt(v|c)`（由 logits 升序决定），然后从修正分布 `q^(α)(v|c) ∝ q(v|c) * rt(v|c)^α` 中采样，其中 α 为噪声强度。这能让低排序 token 获得更高权重，提升生成指令的多样性和信息量。
- 重复上述过程，构建一个规模远大于预算的**候选指令库**（N ≫ B）。

**第二阶段：基于核边界不确定性（KBU）的数据策展**
- **语义不确定性量化**：对候选指令库中的每条指令 `x`，采样 `m` 个回答，通过文本嵌入（如 text-embedding-3-small）计算语义相似度，构建 RBF 核矩阵 `K`。
- **似然加权**：为避免低概率、离题的生成膨胀不确定性，对每个回答计算长度归一化的对数似然，转换为归一化权重 `w̃_i`，形成加权矩阵 `W`，得到单位迹矩阵 `A = W K W / tr(W K W)`。
- **KBU 分数**：计算 `A` 的 Rényi-2 熵：`U_KBU(x) = -log tr(A²)`。该分数反映有效语义模式的数量。
  - 分数低：回答集中于单一模式（太简单或太自信的错误）。
  - 分数过高：回答极度分散，可能超出模型可行动的知识边界。
  - **中间值**表示存在多个可信回答，指示该指令位于模型的**语义知识边界**，是最有价值的学习信号。
- **选择**：根据 KBU 分数百分位范围 `(u_min, u_max)` 挑选指定预算数量的指令进行人工或强模型验证标注，加入训练集。

### 主要结论与发现
- 模型崩溃在指令微调中表现为 **“能力极化”**，而非常见的均匀退化，这为数据策展提供了行动方向。
