## 摘要
关于自改进智能体运行时的安全声明几乎总是自我评级的：一个策略文件、一个护栏或一个README承诺。我们描述了可证伪的发布门槛，以及构建和验证此类系统的方法论，使得任何新能力在发布前必须通过预先指定的、机器可验证的验收套件，并且在每个门槛处保持一组固定的不变性条件。我认为我们已在开源运行时Antahkarana中应用了该方法，通过七个门槛，从基础可观测性进入一个自治理循环，该循环会建议对其自身策略进行更改。任何动作在没有由控制环铸造的安全关键属性能力令牌的情况下都不会到达效应器；该令牌会在一个有限模型的已记录可达状态空间（超过一百万条）上进行彻底的机器检查，并根据执行轨迹重新检查。故意破坏的模型会给出最短的反例，因此检查器具有威力，显而易见。自增强循环受到正向约束：整个写入表面是策略规则，收紧的更改可以自动应用，而放宽的更改始终需要人工合并，并且那些错误预测自身效果的提案会自动关闭。我们发布了所有七个门槛的验收测量结果，精确定义了每个声明的范围（协调骨架的有限模型，而非学习组件），并释放了运行时、命令行工具及门槛套件，因此结果可复现，且门槛可针对其他智能体框架运行。审阅者可以在几秒钟内重复执行那个不可绕过的单一中央命令。

## Abstract
Safety claims on self-improving agent runtimes are almost always self-graded: a policy file, a guardrail, or a README commitment. We describe falsifiable release gates, and a methodology to build and validate such systems, such that every new capability must pass a pre-specified, machine-verifiable acceptance suite before it ships, and a fixed set of standing invariants is preserved at each gate. ## 1. 论文的核心问题与整体含义

- **核心问题**：自改进智能体系统（能修改自身行为）的安全声明，目前几乎完全依赖**自我评估**（如一个策略文件、一个护栏组件或 README 中的承诺）。对于“每一次新能力的获得是否依然保持着上一版本的安全性”这一关键问题，缺乏**外部可验证、可证伪的客观机制**。
- **研究动机**：随着智能体运行时获得记忆、工具访问、多智能体协调以及修改自身策略的能力，单次审计无法保障持续安全。需要一种**将安全作为过程而非一次性承诺**的方法论，且该方法必须可被机器检查、可被证伪。
- **整体含义**：提出**可证伪发布门槛**的概念，将每一次能力进化视为一个“门”，必须通过一套**预先声明、机器可验证的验收套件**才能发布，从而确保核心安全不变性在系统自我演进中持续保持。

## 2. 论文提出的方法论

### 2.1 核心思想：可证伪发布门槛
- **门槛定义**：一个全新的能力在被正式承认存在之前，必须先通过一个预先写好的、可机器判决的验收套件（pass/fail），该套件是**公开且不可事后修改**的。
- **不变性护驾**：整个系统维持六个固定的**不变性**，每个门槛都必须保持它们不被破坏：
  1.  **单一路径**：所有真实动作必须经过控制环裁决，裁决通过后铸造**能力令牌**，效应器无令牌不执行。
  2.  **单调收紧**：系统可自主收紧策略，但**不可自主放宽**；放宽必须经过人工合并。
  3.  **全量审计**：所有决策（模型或人类）产生哈希链记录。
  4.  **哈希载荷**：治理层仅基于内容的哈希做决策。
  5.  **可选零开销**：不使用的安全能力不引入性能负担。
  6.  **门槛先于代码**：验收套件先编写，功能后实现。
- **进阶阶梯**：七个门槛按强制依赖顺序排列（G7 → G12），前一个门槛为后一个提供安全基础。

### 2.2 关键技术实现细节（以 `Antahkarana` 运行时为例）
- **架构分解**：借鉴“内感官”模型，将系统拆分为四个不可绕过的职能模块：
  - `Manas`（感知）：产生新颖性/风险分数。
  - `Buddhi`（裁决）：控制环，给出分层判决并**铸造一次性能力令牌**。
  - `Ahamkāra`（边界）：将令牌绑定到特定身份与租户，实现隔离。
  - `Citta`（记忆）：提供重放、校准与审计线索。
- **安全核心的机器验证（针对 G12）**：
  - **有界模型检查**：对协调骨架（tick、判决、令牌、效应器）的状态空间做彻底枚举（规模 291 个可达状态），验证三个不变性：
    - `INV-1`：无令牌则效应器不执行。
    - `INV-4`：上级在一个 tick 内暂停/终止后，子节点零动作。
    - `INV-5`：能力令牌一次一用。
  - **“牙齿”机制**：为每个不变性故意构造**故意破坏的模型**，要求检查器必须**发现并给出最短反例**，以证明检查器本身有效。
  - **轨迹一致性**：将 100 万条合成合法执行轨迹在模型上重放，确保模型与代码不脱节。
- **自改进的安全约束（G12）**：
  - **写表面受限**：自改进模块只能生成**策略规则变更**（如阈值调整），无法触及裁决系统本身，从类型系统上杜绝越权修改。
  - **收紧自动，放宽人工**：通过策略格自动判定变更方向，**收紧的变更可自动生效，放宽的变更强制人工合并**。
  - **预测自我**：自改进建议还需通过旧门的重放预测验证，若预测效果与实际不符则**自动关闭**。

## 3. 实验设计

- **数据集与场景**：

  - **G9（学习不漂移）**：在**三个保留重放语料库**上评估动态校准的门限，对比静态默认门限的表现。
  - **G10/11（舰队治理与租户隔离）**：模拟子回路暂停/隔离/终止场景，以及跨租户读写的密码学隔离。
  - **G12（自治理）**：通过模型检查验证 291 个状态空间，并用 100 万条合成轨迹做一致性检查。

## 6. 论文的主要结论与发现

- 提出并实例化了**可证伪发布门槛**方法论，将自改进系统的安全从一个“承诺”变成一个**可机器裁决、可复现的过程**。
- 在一个名为 `Antahkarana` 的开源运行时中，**七个门槛全部按预定指标通过**，包括从对抗注入拦截、无漂移学习到自治理循环。
- 通过**有界模型检查**证明安全核心的不可绕过性，并通过“牙齿”机制**显式地展示了检查器能够捕获错误**，增强了验证可信度。
- 设计出**受限的自改进**方案：系统仅能修改策略规则，且只能自动收紧，不能自动放宽，并将错误预测的提案自动驳回。
- 所有代码、门槛套件与运行数据**开源**，使结果可复现，并可作为其他智能体框架的安全检验工具。

## 7. 优点

- **理念先进**：将安全从静态评估提升为**持续、可证伪、自动验证的演化过程**，非常适合自改进系统。
- **强迫纪律**：强制“**门槛先于代码**”和“**牙齿机制**”，确保了验证逻辑本身的有效性。
- **架构清晰**：“内感官”分解与能力令牌机制，提供了**简洁且可证明的单一路径执行流**。
- **正向约束**：自改进的“只能收紧，不能自行放宽”设计，在赋予系统灵活性的同时严格限定了风险边界。
- **高度可复现与可迁移**：提供一套可独立运行的命令行工具和门套件，任何框架都可以接入测试，且关键验证（`atk verify --teeth`）仅需一条命令即可在短时间内完成。

## 8. 不足与局限

- **有界证明而非无界**：机器验证限于**小范围的状态枚举**，并非完全的归纳证明，不能保证在更大规模下无漏洞。
- **模型覆盖范围窄**：验证仅针对**离散的协调骨架**（tick、令牌逻辑），不包括学习组件（如风险评分函数），这部分仍是未验证的薄弱环节。
- **当前结果自评**：所有通过的数据均来自作者自己的测试套件，缺乏**独立的第三方验证结果**，论文也坦诚将此列为主要威胁。
- **运行时监控不完整**：运行时监控采用采样方式而非完整事件观察，仅能缩小但未完全消除模型与实现之间的证明差距。
- **缺乏横向比较**：没有与其他主流智能体安全框架（如 NeMo Guardrails 等）在相同威胁模型下进行实用性或安全性比较。
- **应用环境单一**：当前部署形态为树状网格，未展示更复杂的图状多组织联盟或对抗性监管场景。
