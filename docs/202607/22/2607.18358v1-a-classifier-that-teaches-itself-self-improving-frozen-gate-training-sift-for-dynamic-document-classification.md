## 摘要
文档分类在实验室环境中是已解决的问题，但在企业应用中尚未解决。阻碍通常不在于模型架构；而在于必须先于模型进行的标注项目，以及机构对于模型存在后让其自我再训练的恐惧。我们提出了SIFT（自改进、冻结门训练），一种动态分类器服务，旨在解决这两个问题。SIFT通过一个特意设计的廉价、基于CPU的流水线提供分类服务，使用SPLADE稀疏编码器馈送给LightGBM头，并仅将低置信度的少数页面升级至LLM裁判。裁判的判决被写回到标注语料库中，因此昂贵的模型持续教导廉价的模型：升级率下降，语料库从生产流量中增长而非依靠前期标注工作，并且准确度随着使用而累积。引入新的文档家族仅需一个声明式包、标签空间、锚短语和裁判词汇表，而非一个标注项目。更难的问题是安全性：自主再训练的分类器可能无声地退化。SIFT通过两部分的推广门来解决此问题，包括关键标签F1回归检查加上一个冻结的黄金回归集（模型从未在其上训练），任一检查不合格都会否决推广。这使得“每月无需人工再训练”从鲁莽行为变为常规操作。我们描述了架构、自供给语料循环、冻结门推广机制，以及一个说明性的多领域部署，并讨论了边际标注成本趋于零的分类器的经济性。

## Abstract
Document classification is a solved problem in the laboratory and an unsolved one in the enterprise. The blocker is rarely model architecture; it is the labeling project that must precede a model and the institutional fear of letting a model retrain itself once one exists. We present SIFT (Self-Improving, Frozen-gate Training), a dynamic classifier service, which attacks both. SIFT serves classification from a deliberately cheap, CPU-bound pipeline, a SPLADE sparse encoder feeding a LightGBM head, and escalates only the low-confidence minority of pages to an LLM judge. The judge's verdicts are written back into a labeled corpus, so the expensive model continuously teaches the cheap one: the escalation rate falls, the corpus grows from production traffic rather than from an up-front annotation effort, and accuracy compounds with use. Onboarding a new document family requires only a declarative bundle, label space, anchor phrases, and a judge glossary, not a labeling project. The harder problem is safety: an autonomously retraining classifier can silently regress. SIFT resolves this with a two-part promote gate, a critical-label F1 regression check plus a frozen golden regression set the model is never trained on, either of which vetoes promotion. This turns "retrain monthly without a human" from reckless into routine. We describe the architecture, the self-feeding corpus loop, the frozen-gate promotion mechanism, and an illustrative multi-domain deployment, and we discuss the economics of a classifier whose marginal labeling cost trends toward zero.

### 2. 方法论

- **核心思想**：采用“廉价模型处理常见样本，昂贵模型仅处理罕见低置信度样本”的成本级联架构，并利用昂贵模型的输出持续教导廉价模型，实现自举、自改进。同时设计一个“冻结门”机制保障自动再训练的安全性。
- **关键技术细节**：
  - **廉价模型**：SPLADE稀疏编码器将页面转换为词法特征，结合锚短语、正则匹配计数和长度特征，输入LightGBM分类头，输出标签和置信度。整个过程基于CPU，毫秒级响应。
  - **昂贵模型（LLM裁判）**：仅当廉价模型置信度低于阈值时，页面才会被递交给LLM裁判进行推理。裁判接收声明式包中的允许标签、词汇表和策略，给出高置信度判决。
  - **自供给语料循环**：
    - 裁判的判决（高于最低置信度）以及人工审核修正的结果会被写回标注语料库，作为新的训练数据。
    - 引导路径支持从零标注数据开始：将原始未标注页面批量送入裁判，利用其高置信度判决构建初始语料库和模型。
    - 主动学习队列将最难判别的页面（裁判置信度最低）优先呈现给人工审核，确保稀缺的专家精力投入在最需要的地方。
  - **冻结门推广机制**（安全自动再训练的核心）：
    - 每个候选模型必须通过两个检查才能被推广（取代线上模型）：
      1. **关键标签F1回归检查**：声明式包中预定义的关键类别（如“NDA”）的F1值不得低于当前线上模型，否则推广被阻止并返回差值。
      2. **冻结黄金回归集检查**：一个在训练时完全不参与、不可变的小型黄金标注集，用于评估候选模型。如果在该集合上关键标签F1或整体准确率下降，则同样否决推广。
    - 调度器可根据新增行数、时间间隔或数据漂移触发自动再训练。
    - 只有通过双门检查的候选模型才会自动取代线上模型；否则将停留在待审队列，需人工介入。
  - **领域表达**：新文档家族通过一个声明式YAML包定义，包含标签空间、每类锚短语、裁判词汇表/策略、关键标签列表、黄金回归集等，无需编写代码或进行单独标注项目。
- **算法流程**（文字说明）：
  1. 页面文本经过SPLADE编码和特征构造，由LightGBM预测类别和置信度。
  2. 若置信度 >= 阈值，直接返回结果；否则，将文本和领域包信息发送给LLM裁判，裁判根据策略给出标签。
  3. 裁判结果若满足置信度要求，被写回语料库（标注来源标记为“llm”）。
  4. 人工审核过的页面（经主动学习队列）写回语料库（来源标记为“human”）。
  5. 调度器根据触发条件（如新增200行、漂移超过0.08）启动训练，生成候选模型。
  6. 候选模型在冻结黄金集上评估，并进行关键标签F1与当前模型对比。两个检查均通过，则自动推广，并记录审计日志；任何一项失败，模型保留在待审队列。

- SIFT通过将生产流量转化为持续的廉价标注来源，消除了前期标注项目的需求，使得分类器的边际标注成本趋近于零。

### 7. 优点

- **系统性创新**：将成本级联、主动学习、弱监督和模型安全更新有机整合为一个可部署的自改进服务，直面生产环境痛点。
