## 摘要
近年来，LLM 智能体处理的任务日益复杂、开放，而外部技能——提供给智能体的可重用程序性知识——进一步扩展了这种能力。然而，固定且手工编写的技能很少是最优的，无法适应智能体遇到的任务多样性。自我改进的智能体通过从执行轨迹重写自己的技能文件来解决这一问题，在挑战性基准上取得了显著提升。但这类自我演化仍是非递归的：它只改进了任务技能（智能体做什么），而改进过程（它如何改进）则是一次性编写并保持固定。我们提出了 MetaSkill-Evolve，一个双时间尺度框架，使智能体的技能改进递归化：每个分支同时携带一个任务技能 s 和一个分支局部的元技能 m=(ψ,σ,α,π,ε)，其五个分量分别参数化了改进流程中的分析器、检索器、分配器、提议器和演化器智能体。任务技能在快速循环中演化，而元技能则在较慢的循环中通过应用于自身的同一流程演化，无需额外模型或目标。在五个流程智能体共享一个冻结基座模型的情况下，MetaSkill-Evolve 在三个智能体基准（OfficeQA、SealQA、ALFWorld）上优于无技能、静态技能和单层演化基线，分别在留出测试准确率上相对原始基座模型提升了 +23.54、+16.09 和 +1.92 点。

## Abstract
Recent LLM agents tackle increasingly long-horizon, open-ended tasks, and external skills, reusable procedural knowledge supplied to the agent, further extend this capability. However, a fixed, hand-authored skill is rarely optimal, and cannot adapt to the diversity of tasks an agent encounters. Self-improving agents address this by rewriting their own skill files from execution traces, yielding meaningful gains on challenging benchmarks. Yet such self-evolution remains non-recursive: it improves only the task skill (what the agent does) while the improvement procedure (how it improves) is authored once and held fixed. We introduce MetaSkill-Evolve, a two-timescale framework that makes agentic skill improvement recursive: every branch carries both a task skill $s$ and a branch-local meta-skill $m=(ψ,σ,α,π,\varepsilon)$ whose five components parameterise the Analyzer, Retriever, Allocator, Proposer, and Evolver agents of the improvement pipeline. Task skills evolve on a fast loop while the meta-skill evolves on a slower one under the same pipeline applied to itself, with no additional model or objective. With all five pipeline agents sharing a single frozen backbone, MetaSkill-Evolve outperforms no-skill, static-skill, and single-level evolution baselines on three agentic benchmarks (OfficeQA, SealQA, ALFWorld), improving held-out test accuracy over the raw backbone by +23.54, +16.09, and +1.92 points respectively.

### 2. 论文提出的方法论：核心思想、关键技术细节与算法流程

*   **核心思想 (MetaSkill-Evolve)**：设计一个**双时间尺度演化框架**。每个演化分支的状态由一个三元组 `b = (s, m, h)` 定义，其中：
    *   `s`: 任务技能（Target Skill），定义“做什么”。
    *   `m`: **分支局部的元技能 (Meta-Skill)**，定义“如何改进”，由五个可学习组件构成：`m = (ψ, σ, α, π, ε)`。
    *   `h`: 分支的迭代历史。

*   **元技能的五个组件**：每个组件本身都是一个与任务技能格式相同的 Markdown 程序文件，分别参数化改进流程中的一个专用智能体：
    *   **`ψ` (Analyzer, 分析器)**：诊断失败轨迹，输出错误标签 `φ` 和自由文本分析 `a`。
    *   **`σ` (Retriever, 检索器)**：根据错误标签 `φ`，检索本分支或其他分支的相关成功/失败案例作为灵感 `I`，实现跨分支经验共享。
    *   **`α` (Allocator, 分配器)**：根据当前分支的停滞程度，动态决定本次迭代生成的子代预算 `K`，在广度和深度之间取得平衡。
    *   **`π` (Proposer, 提议器)**：基于失败案例、分析 `a` 和灵感 `I`，提出具体的技能编辑方案 `δ`。
    *   **`ε` (Evolver, 演化器)**：将编辑 `δ` 应用到磁盘的技能文件上，并进行一致性验证。

*   **双时间尺度机制**：
    *   **快速循环 (任务技能演化)**：在每次迭代中，选定一个前沿父节点，运行的五个智能体管道（由父节点的元技能 `mv` 参数化）来改进其任务技能 `sv`，生成多个子代技能并评估其效用增益 `ΔU`。
    *   **慢速循环 (元技能演化)**：每 `H` 次快速迭代触发一次。它首先计算过去 `H` 个子代的**元生产力（Meta-Productivity）**`P(m|s)`——即这轮快速循环产生的平均效用增益。然后，将这段改进历史打包成一个“元失败轨迹”，**重新调用同样的五个智能体管道，但将操作目标从任务技能文件切换为元技能文件**。通过对元技能组件 `(ψ, σ, α, π, ε)` 的分析、检索、分配、提议和执行，实现元技能自身的进化。这是一次有界的一级递归。

*   **前沿选择 (Frontier Selection)**：演化过程被记录为一个持久化的有向无环图（DAG）。每次迭代从存档（由严格改进的节点组成）中选择下一个要扩展的父节点，其得分函数为：
    `Score(v) = η1 * Uv + η2 * Pv + η3 * Nv`
    其中，`Uv` 是节点效用（任务性能），`Pv` 是元生产力（改进潜力），`Nv` 是新颖性（鼓励探索未被充分选择的分支）。这种组合权衡了利用、优化轨迹质量和探索多样性。

### 6. 论文的主要结论与发现
*   **递归自我改进的范式可行且有效**：MetaSkill-Evolve 首次证明了，通过双时间尺度演化，智能体改进自身的流程（元技能）是可行且能带来显著收益的。
*   **性能显著提升**：与原始基座模型相比，在 OfficeQA、SealQA 和 ALFWorld 上的留出测试准确率分别**提升了 23.4、16.1 和 1.9 个百分点**。所有设计选择（添加技能→演化任务技能→演化改进流程）都带来了独立收益。

*   **组件分工明确**：在不同任务领域，元技能的各个组件贡献度不同。例如，OfficeQA 中动态分配器（`α`）最关键，而 SealQA 和 ALFWorld 中编辑提议器（`π`）占主导，表明不同场景需要不同的改进策略。

### 7. 优点：方法或实验设计上的亮点
*   **概念创新性强**：将自我改进从“优化任务策略”提升到“优化改进策略本身”的层面，实现了一种简洁优雅的递归自我改进。
*   **实现成本低，通用性高**：元技能与任务技能使用完全相同的 Markdown 格式和同一套智能体管道，无需引入新模型、新架构或新目标函数，可轻易集成到现有框架。
*   **前沿选择机制巧妙**：结合了效用、元生产力和新颖性三种指标，优化了演化搜索的平衡性，防止搜索陷入停滞或早期局部最优。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等
*   **固定管道架构**：元技能可以演化，但由五个智能体（Analyzer, Retriever 等）组成的管道结构本身是固定且硬编码的。框架不能演化出新的智能体角色或改变他们之间的连接关系。
*   **慢速循环超参数固定**：元技能的更新频率 `H` 被设为固定值。虽然扫参实验找到了一个合适默认值，但自适应、动态调整的更新策略可能更优。
*   **基准测试范围**：评估局限于三个特定 Agent 基准，研究对象主要是文本问答和虚拟环境交互。对于具有更嘈杂反馈的真实世界、长周期任务，其效果和鲁棒性尚未得到验证。
*   **递归深度限制**：这是一次只有“一层”的有界递归。框架仅让元技能自我改进，而元元技能的演化（即如何改进“元技能”的过程）并未实现。
*   **性能天花板明显**：在基座模型本已表现卓越的任务（如 ALFWorld）上，尽管有相对提升，但绝对提升空间有限（+1.92%），说明该方法的价值在很大程度上取决于基座模型存在可被改进的“短板”。
