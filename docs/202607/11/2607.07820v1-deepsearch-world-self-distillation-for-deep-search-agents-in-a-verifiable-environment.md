## 摘要
训练工具使用代理从自身经验中改进仍具挑战，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供弱监督。我们提出了DeepSearch-Evolve，一个建立在DeepSearch-World之上的网络代理自蒸馏框架，DeepSearch-World是一个确定性、可验证的环境，具有可复现的搜索和页面阅读工具。DeepSearch-World包含42万个由实体级随机游走构建的多跳问答任务，并支持对自进化有用的关键代理认知行为，包括进度验证、基于依据的反思和失败恢复。DeepSearch-Evolve迭代执行轨迹生成、过滤、数据混合和微调来训练更强的代理。在没有从更强大模型蒸馏的情况下，DeepSearch-World-9B取得了与开源代理相比有竞争力的表现，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能够为长程网络代理实现可扩展的自进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来关于自改进深度搜索代理的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

*   **核心问题**：如何让能够执行长期、多步骤任务的“深度搜索”网络代理，从自身经验中持续、稳定地自我改进（Self-evolve）？
*   **研究背景与动机**：
    *   **现有方法的局限**：
        1.  **基于模仿的监督微调（SFT）**：严重依赖固定的、由更强模型生成的“教师”轨迹，导致代理性能受限于“教师”模型能力上限，且轨迹多样性不足，容易饱和。
        2.  **基于稀疏奖励的强化学习（RL）**：只提供最终结果的奖励信号，对于需要多步交互的长程任务，学习过程极不稳定，难以定位具体失败环节（是查询不当、工具选择错误还是证据提取失败）。
        3.  **在线策略自蒸馏（On-policy Self-Distillation, OPSD）**：虽然提供了更稠密的步骤级监督信号，但需要一个确定且可验证的环境。在非确定性环境中，教师策略在每一步提供的指导可能充满噪声，难以直接应用于长程代理。
*   **整体含义**：本论文提出，解决上述困境的关键在于构建一个“可验证环境”，并在此基础上设计一种新的自蒸馏框架，使代理能通过验证自身的探索轨迹来获得高质量的步骤级监督信号，从而实现“自我进化”。这项工作展示了从模仿、奖励驱动探索到“可验证的自进化蒸馏”的范式转变。

### 2. 论文提出的方法论

论文的核心方法论由两部分构成：**可验证工具环境 DeepSearch-World**和**自蒸馏框架 DeepSearch-Evolve**。

#### 2.1 可验证工具环境：DeepSearch-World

*   **核心思想**：构建一个确定性的、可复现的离线环境，替代真实互联网，为代理的训练和评估提供稳定、可验证的交互体验。
*   **关键技术细节**：
    1.  **数据与任务构建**：
        *   **知识图谱**：基于维基百科的超链接图 `G = (V, E)`。
        *   **多跳问答生成**：通过实体级随机游走 `τ = (v1, ..., vH)` 生成推理链。将链上的实体进行模糊化处理，构造出需要多步推理才能回答的复杂问题。
        *   **规模**：构建了包含**42万个**多跳问答任务的训练池，和一个用于验证的**377个**高质量样本集 `DeepSearch-Val`。答案由至少三位专家独立验证。
    2.  **离线工具集**：
        *   **搜索工具（`web_search_wiki`）**：基于Lucene BM25的确定性检索函数。输入自然语言查询，输出排名靠前的文章标题、简短摘要和确定性URL，模拟搜索引擎结果页。
        *   **浏览/访问工具（`visit_wiki`）**：确定性阅读函数。通过SQLite偏移索引，将URL直接映射到离线语料库中的完整维基百科文章，模拟浏览器访问。
    3.  **环境内嵌的反思机制**：
        *   **进度验证**：环境存储每条问题的目标实体集 `Ti`。在代理交互过程中，环境会判断工具调用是否命中未解决的实体，实现**步骤级**的进度追踪，无需昂贵的LLM评判。
        *   **失败反馈**：对失败的搜索/访问调用，环境会触发阶段性、基于规则的反思信号，引导代理进行查询改写和重试，生成“搜索-失败-反思-重试”的含错修正轨迹。

#### 2.2 自蒸馏框架：DeepSearch-Evolve

*   **核心思想**：通过一个迭代循环，让代理在可验证环境中扮演“教师”和“学生”双重角色，将从“教师”轨迹中提取的高质量步骤级监督信号，内化到“学生”的ReAct策略中。
*   **算法流程与关键步骤**：
    1.  **脚手架式教师轨迹生成（Scaffold Teacher）**：
        *   代理分**Plan**, **Act**, **End**三个阶段运作。
        *   **Plan**：初始化一个结构化的**进度状态**，包含 `completed_list`, `todo_list`, `experience`, `information` 四个字段，用于显式追踪任务分解、证据和失败经验。
        *   **Act**：迭代调用工具，根据环境反馈和进度状态更新工作记忆。
        *   **End**：在所有证据充足后，根据已验证的工作记忆生成简洁的最终答案，减少幻觉。
    2.  **轨迹过滤与转换**：
        *   **拒绝采样与质量过滤**：对生成的轨迹，首先使用LLM（Qwen3.5-9B）判断答案是否正确。然后进行轨迹级质量过滤，去除冗余证据、目标对齐差或推理不一致的轨迹，只保留高质量的成功轨迹。
        *   **脚手架到ReAct的格式转换**：将包含Plan/Act/End阶段和结构化状态的“教师”轨迹，**转换**为标准ReAct格式的监督数据。具体做法是，将进度状态摘要(`Pt`)、反思信息(`Rt`)和局部推理(`At`)重写并注入到 `<think>...</think>` 块中，而工具调用和观测保持不变。这保障了“学生”代理是一个可独立部署的标准ReAct代理。
    3.  **监督式自蒸馏更新（Evolving SFT）**：
        *   **迭代训练循环**：在第 `R` 轮，当前模型 `πθR` 作为“教师”生成轨迹。这些轨迹经过滤和格式转换后，用于微调“学生”模型 `πθR+1`，而该“学生”模型即为下一轮的“教师”。
        *   **训练目标**：本质上是硬标签的令牌级策略蒸馏，最小化负对数似然损失。
        *   **重要性采样混合**：训练采用异步机制，当新生成的轨迹多于SFT更新所需时，采用衰减因子（`γ=0.5`）的重要性采样策略混合不同轮次的数据，优先学习近期的高质量经验，同时缓解灾难性遗忘。

### 6. 论文的主要结论与发现

    *   **轨迹质量过滤**，特别是对**答案正确性**的验证，是自蒸馏最关键的保障。
    *   将**反思和结构化规划**内化到ReAct思考轨迹中，对于提升标准代理格式的性能至关重要。

### 7. 优点

*   **范式创新**：将自进化问题与环境构建紧密结合，提出了“可验证环境+自蒸馏”的范式，为解决长程代理的训练难题提供了新思路。

### 8. 不足与局限

*   **领域局限性**：当前可验证环境完全构建于**维基百科**之上，其任务的多样性和知识的广度受限。面对真实网络中动态、多领域、结构复杂的内容，模型的泛化能力尚待考证。
*   **蒸馏方法单一**：自蒸馏更新仅采用了进化式SFT。论文也指出，将其与RL式或OPSD式的更新结合可能进一步提升效果，但在注入规划、纠错等高级策略方面仍待探索。
*   **跨语言迁移有限**：尽管在中文基准上有提升，但由于训练数据为英文，其跨语言性能（如BrowseComp-ZH）仍有较大提升空间。
*   **潜在的偏差风险**：训练数据基于维基百科，模型可能会继承其中蕴含的社会偏见。
*   **工程复杂性**：整个框架涉及环境构建、多阶段轨迹生成、过滤、格式转换和异步训练，对工程实现要求较高。
