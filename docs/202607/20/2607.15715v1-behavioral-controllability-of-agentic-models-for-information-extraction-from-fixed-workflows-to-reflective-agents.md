## 摘要
大语言模型智能体越来越多地用于复杂的信息抽取任务，但反思和记忆等智能体组件相比固定的大语言模型工作流是否能带来可观察且可控的改进，目前尚不清楚。我们通过会议论文数据集抽取任务研究此问题，在此任务中系统需识别学术PDF中提及的数据集并生成结构化记录。我们比较了固定工作流基线与反思型智能体变体，并指定一种优化的智能体条件（S2），后者通过更丰富的PDF工具和动态工具选择扩展了相同任务。我们的评估侧重于过程层面行为——包括工具执行、重试、反思、记忆使用、运行时间和故障恢复——而将抽取覆盖率和字段完整性作为次要结果指标。本文描述了智能体机制何时改变系统行为，这些变化是否改进了任务完成情况，以及观察到的故障模式如何在相同的评估框架下激发优化智能体的设计。

## Abstract
Large language model (LLM) agents are increasingly used for complex information-extraction tasks, yet it remains unclear whether agentic components such as reflection and memory lead to observable and controllable improvements over fixed LLM workflows. We study this question through conference-paper dataset extraction, where a system must identify datasets mentioned in scholarly PDFs and produce structured records. We compare a fixed workflow baseline with reflective agent variants and specify an optimized agent condition (S2) that extends the same task with richer PDF tools and dynamic tool selection. Our evaluation emphasizes process-level behavior--including tool execution, retries, reflection, memory use, runtime, and failure recovery--while treating extraction coverage and field completeness as secondary outcome measures. The paper characterizes when agentic mechanisms change system behavior, whether these changes improve task completion, and how the observed failure modes motivate an optimized agent design under the same evaluation harness.

## 论文核心问题与整体含义
随着大语言模型（LLM）被广泛嵌入智能体系统，如何评估反思（reflection）、记忆（memory）等智能体机制在信息抽取中的实际价值成为一个开放问题。本文以学术会议论文中的数据集抽取为场景，探究从固定工作流到反思型智能体的“行为可控性”——即系统的决策过程是否可观察、可配置、可复现且可比较。论文的核心不在于追求绝对抽取精度，而在于揭示智能体机制何时、如何改变系统行为，以及这些变化是否能转化为更优的任务完成度与故障恢复能力。

## 方法论
论文设计了一个分层的系统框架，将同一抽取任务分别实现为：
- **S0（固定工作流）**：无反思、无记忆、无重试，按预设顺序进行PDF解析、文本提取、LLM元数据抽取、数据集名抽取和结构化记录写入。
- **S1a/S1b（反思型智能体）**：基于ReAct模式的智能体，包含“观察→思考→行动→反思→学习→调整”循环。工具管理器注册了PDF解析、元数据抽取、数据集名抽取、数据集详情抽取四个工具。反射模块对工具输出评分，若低于阈值（0.6）且存在诊断性问题，则在重试预算内重试。记忆模块存储短期和长期经验，并可注入后续提示。S1a仅用规则反射，S1b额外启用LLM反射。
- **S2（优化智能体，仅设计未运行）**：将固定高层计划替换为基于文档状态的动态规划器，并扩展为12个原子工具（6个PDF工具、3个LLM工具、3个证据接地工具），以解决链接发现、章节回退等故障模式。

评估框架的特点在于过程中心（process‑centric），除了最终JSONL记录数、字段完整度等结果指标，更关注日志中的工具调用、反射事件、重试决策、记忆注入和故障恢复等行为信号。

- **对比方法**：S0（无智能体）、S1a（规则反射智能体）、S1b（规则+LLM反射智能体）以及作为未来设计的S2。所有条件共享相同的语料、输出模式、LLM后端（vLLM部署的openPangu‑Embedded‑7B）和重试预算（最多2次）。

## 主要结论与发现
- **行为变化明显**：智能体外壳使系统产生大量过程级信号（反射、重试、记忆注入），日志体积远超固定工作流，表明智能体机制使得行为更可观察、可配置。
- **结果提升有限**：S1a和S1b分别产生165、168条记录，略高于S0的158条，但链接率始终低于20%，链接接地仍是瓶颈。
- **反射价值主要体现在诊断而非数量**：LLM反射相比规则反射仅多3条记录，但能提供更具体的故障诊断和优化方向（如建议搜索特定章节）。
- **故障模式驱动优化设计**：案例中暴露的凭据接地弱、记忆噪声、工具单一等问题，直接催生了S2的12原子工具和动态规划需求。
- 最终结论强调智能体评估应以过程可控性为核心，而非仅比较最终输出量。

## 优点
- **过程中心评估视角新颖**：在智能体评估领域率先将“行为可控性”作为独立评价维度，提出可观察性、可配置性、可复现性和可比较性四项具体标准。
- **可控实验设计**：固定语料、模式、模型和运行参数，在不同条件间仅改变智能体部件，提升了因果归因的清晰度。
- **深入的案例剖析**：通过日志回溯解释正常执行、有界重试和LLM诊断三种典型行为，展示了智能体机制的实际作用机制与失败原因。
- **从诊断到设计的闭环**：基于S1的失败模式驱动了S2的详细设计（12个原子工具、动态规划器），体现了研究的前瞻性和工程指导价值。

## 不足与局限
- **语料与规模受限**：仅使用NeurIPS 2024且每次运行50篇论文，模型也固定为openPangu‑Embedded‑7B，外部效度不足。
