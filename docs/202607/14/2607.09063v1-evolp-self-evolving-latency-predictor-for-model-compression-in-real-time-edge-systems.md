## 摘要
边缘设备越来越多地用于在嵌入式系统上部署深度学习应用。许多应用的实时性和边缘设备的有限资源要求进行面向延迟的神经网络压缩。然而，在真实设备上测量延迟既困难又昂贵。因此，本文提出了一种新颖高效框架EvoLP，用于准确预测模型在边缘设备上的推理延迟。该预测器能够在网络压缩过程中演化以获得更高的延迟预测精度。实验结果表明，通过在三种边缘设备和四种模型变体上的评估，EvoLP优于先前最先进的方法。此外，将其整合到模型压缩框架中时，它能有效引导压缩过程，在满足严格延迟约束的同时实现更高的模型精度。我们在 https://github.com/ntuliuteam/EvoLP 上开源了 EvoLP。

## Abstract
Edge devices are increasingly utilized for deploying deep learning applications on embedded systems. The real-time nature of many applications and the limited resources of edge devices necessitate latency-targeted neural network compression. However, measuring latency on real devices is challenging and expensive. Therefore, this letter presents a novel and efficient framework, named EvoLP, to accurately predict the inference latency of models on edge devices. This predictor can evolve to achieve higher latency prediction precision during the network compression process. Experimental results demonstrate that EvoLP outperforms previous state-of-the-art approaches by being evaluated on three edge devices and four model variants. Moreover, when incorporated into a model compression framework, it effectively guides the compression process for higher model accuracy while satisfying strict latency constraints. We open source EvoLP at https://github.com/ntuliuteam/EvoLP.

## 1. 研究动机与背景
- **核心问题**：在资源受限的边缘设备上部署深度神经网络时，实时应用要求严格的推理延迟。模型压缩能提升效率，但需要以真实延迟作为约束，避免反复进行昂贵的物理设备延迟测量。
- **现有方法的不足**：
  - 查找表（LUT）无法预测新模型；硬件模拟器难以构建且不通用。
  - 回归/学习方法（如nn-Meter、TALM）需要大量采样与测量，构建时间可达数天，尤其是在模型压缩过程中，层配置持续变化导致搜索空间极大（约 1.3 × 10¹⁰），难以承受。
- **目标**：提出一个高效、可自进化的延迟预测器，能在压缩过程中逐步提升预测精度，以替代昂贵的实机测量。

## 2. 方法论
### 2.1 核心思想
- EvoLP由三个阶段构成：**模型采样**、**延迟预测器生成**、**预测器自进化**。
- 将搜索空间限制在待压缩模型本身，大幅降低采样成本；通过进化矩阵在压缩过程中自动学习设备运行时特性，使预测器越用越准。

### 2.2 关键技术细节
- **模型采样（Layer Pool构建）**
  - 将模型层分为**常量层**（压缩过程中延迟变化可忽略）和**可变层**。
  - 仅对可变层按配置范围进行采样，生成数千个单层模型，形成“层池”。
  - 在真实设备上测量这些单层模型的延迟，整个过程可在数小时内完成（对比nn-Meter需数天）。

- **层间延迟预测器（LWLP）生成**
  - 使用轻量三层MLP（多层感知机）学习层配置到延迟的映射，公式如下：  
    `g(x) = σ²(W² σ¹(W¹ σ⁰(W⁰ x)))`
  - 训练出的LWLP能推断任意未见过层配置的延迟。

- **整体模型延迟预测与进化矩阵（EM）**
  - 整体预测：`P = Σ ReLU(Tᵢ + Σ aᵢᵐ * Tᵐᵢ)`  
    - 其中Tᵢ为第i层的预测延迟（常量层直接用实测值），aᵢᵐ为EM中的可训练参数，用于模拟层间交互、数据传输、分支并行等运行时特征。
  - EM采用类似泰勒展开的线性组合，实验表明一阶（k=1）已足够。
  - ReLU用于模拟分支并行。

- **压缩过程中的自进化**
  - 在每一轮压缩训练中，根据重要性因子动态确定压缩比例，并利用当前预测器获得预测延迟p。
  - 同时并行在边缘设备上测量真实延迟r（与训练分离，不干扰）。
  - 获得(r, p)对后，通过随机梯度下降更新EM，使预测不断逼近真实延迟，尤其在延迟约束附近精度逐步提升。
  - 进化过程中还会保留未校正的预测延迟p以维持EM训练稳定性。

## 7. 优点
- **高效构建**：仅需对目标模型采样，搜索空间极小，测量耗时从数天降至数小时。
- **动态进化**：进化矩阵不断吸收压缩过程中的实际延迟信息，无需额外采样，实现了“越用越准”。
- **轻量实现**：MLP预测器和EM更新均非常轻量，不影响压缩训练效率。
- **硬件自适应**：能隐式学习数据搬运、分支并行等难以建模的运行时特性。
- **开源**，易于复现和扩展。

## 8. 不足与局限
- **模型覆盖有限**：仅在VGG、ResNet等四种经典架构上验证，未测试Transformer等新型结构。
- **数据集局限**：压缩后模型的精度测试仅使用ImageNet-100，更大规模数据集上的效果有待验证。
- **在线测量依赖**：自进化仍需在压缩过程中进行少量真实延迟测量（虽可并行），并非完全脱离真实设备。
- **应用场景限定**：主要针对延迟预测，虽提及可扩展到功耗、精度等，但未在本工作中实现。
- **EM线性假设**：用一阶线性组合近似层间交互，在更复杂的硬件调度下可能不够精确。
- 硬延迟约束下的“精确逼近”仍是依赖预测器的误差容忍，极端严格的场景下仍需最终实测。
